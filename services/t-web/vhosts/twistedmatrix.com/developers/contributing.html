Contributing to Twisted

Non-Code Ways to Contribute

While there's always more work to be done on the codebase, Twisted Matrix Labs needs as many people we can get in other roles, too: web designers, system administrators, technical writers, people who write documentation, documentation authors, people to proofread documentation (you get the idea)... and last, but not least, we need users. If you have feedback about Twisted or just think it should do something differently, don't be shy about letting us know.

The more people use the code and find issues with it, the faster we can fix those issues and the more sure we'll be that it's really stable. The more people who can keep the basic services running at twistedmatrix.com and elsewhere, the less time the developers have to spend doing things other than coding and testing.

If you can't contribute in any of these ways, but you appreciate Twisted and want to help the community and future development, consider a monetary donation instead. This is not tax deductable. Donations will be used to further Twisted development.

How To Become a Developer

First, you will want to get in touch with the development team; we highly recommend joining the general mailing list and/or the Twisted-Web mailing list, and reading the archives (general, twisted-web) to get started. You can also get the SVN code, see the SVN page for more information.

Then you'll want to submit a patch. Find something small about Twisted that bugs you and fix it, and share the fix with us, or add a feature that you find particularly useful. It might not be integrated right away, but you're sure to spark some discussion -- we love getting code!

If you want to become a developer, it is important to understand that all your contributions (including those initial patches you send to the bug tracker) will have to licenced under the MIT licence.

Patches should be submitted in diff -u format so they're easy to read.
