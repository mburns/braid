Technical Specifications

Basic Information

Twisted is an asynchronous networking framework written in Python. It has a pluggable main-loop which can run anywhere from poll() on Linux in CPython, to polling emulated with threads on Jython.

In the development of Twisted, we use eXtreme programming (to the extent that one can do so without a single requirements-setting "customer"), so there exist Unit Tests for much of the codebase.

It is designed to interfere as little as possible with existing code; if your application is already event based, then integrating Twisted may be only the work of one afternoon.

Specifications

Currently, we have a specification for the Banana protocol that is part of Twisted Spread. Further specifications will be forthcoming to encourage interoperability at the protocol level with Twisted.

Call for Benchmarks

Twisted anecdotally benchmarks "well enough" but we need more hard numbers. If you have some technical information you want to contribute we'd love to host it here. We also welcome contributions of benchmarking test suites.
